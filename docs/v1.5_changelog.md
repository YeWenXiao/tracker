# v1.5 开发日志 (v1.5-hotswap 分支)

## 核心目标
实现程序不停、不断流的情况下，临时更换要识别的目标。

## 技术方案
- **线程安全**: Python GIL + 原子引用替换
- **触发方式**: 按键 `r` / 目录监控 / HTTP API
- **实时标注**: annotate_live.py 在视频流上直接框选新目标
- **兼容性**: 不加新参数时行为与 v1.0 一致

---

## 更新记录

### Round 1 - 初始开发 (2026-03-17)

**改动内容:**
1. TargetRecognizer 增加 `reload_targets()` 方法
   - 线程安全的原子替换策略
   - 准备好新数据后一次性替换引用
2. 运行时换目标触发方式
   - 按键 `r`: 手动触发重新加载
   - 目录监控: watchdog/轮询自动检测 targets/ 变化
   - HTTP API: Flask/FastAPI 端点接收新目标图片
3. annotate_live.py: 实时视频流上标注新目标
4. 识别结果 overlay 显示目标数量和最后更新时间
5. README 更新

**技术细节:**
- 原子替换: 新建完整 targets 列表后，单次赋值 `self.targets = new_targets`
- HTTP API 用于远程换目标（手机发送新图片）
- 目录 watchdog 用 polling observer 确保跨平台兼容

**预期提升:**
- 换目标零中断（原来需要重启，~2秒中断）
- 支持远程换目标（HTTP API）
- 支持实时框选新目标

**状态:** ✅ Round 1 完成，已push到GitHub

**实际改动统计:**
- recognize.py: 492→549 行（+57行，reload_targets、原子替换、目标监控线程、按键r触发）

**关键技术实现:**
- `_prepare_targets()` 提取为独立方法，返回新列表不修改 self

- 识别线程用 `targets = self.targets` 本地引用避免中途切换
- 目标监控线程每2秒检查 target_info.json 的 mtime
